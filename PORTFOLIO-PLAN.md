# แผนเว็บพอร์ตโฟลิโอ (สรุปก่อนเริ่มสร้าง)

เอกสารนี้สรุปสิ่งที่จะสร้าง โดยอ้างอิงสไตล์จาก https://morgorn-portfolio.vercel.app/ แล้วปรับตามที่คุยกันไว้ ยังไม่ได้ลงมือเขียนโค้ด — อ่านทบทวนก่อน ถ้าโอเคค่อยให้เริ่มสร้างจริง

## เป้าหมาย

เว็บพอร์ตโฟลิโอสำหรับยื่นสมัครงานสาย IT เน้นให้ดูเป็นมืออาชีพ โหลดเร็ว แก้ไขง่าย และพร้อมขึ้น host จริงในอนาคต

## เทคโนโลยี

- **Next.js (App Router) + TypeScript** ตามที่ต้องการ
- **Tailwind CSS** สำหรับดีไซน์
- **ไม่มีระบบหลังบ้าน (backend/database)** — ดูเหตุผลด้านล่าง

### ต้องมี backend ไหม?

**ไม่ต้อง** เว็บพอร์ตโฟลิโอเก็บข้อมูลทั้งหมด (ประวัติ, สกิล, โปรเจกต์) เป็นไฟล์ในโค้ดได้เลย ไม่ต้องมีฐานข้อมูล ไม่ต้องมีระบบล็อกอิน — ต่างจากเว็บตัวอย่างที่มีปุ่ม Sign In/Sign Up และหน้า Contact ที่ต้องล็อกอินก่อนถึงจะกรอกฟอร์มได้ (ส่วนนี้จะ**ไม่ทำตาม** เพราะไม่จำเป็นสำหรับพอร์ตโฟลิโอส่วนตัว)

ถ้าอนาคตอยากได้ฟอร์มติดต่อที่ส่งอีเมลจริง ใช้บริการสำเร็จรูป (Formspree, EmailJS) พอ ไม่ต้องเขียน backend เอง

## สิ่งที่หยิบมาจากเว็บตัวอย่าง

- โครงหน้า: Home (Hero) → About → Skills → Contact
- ปุ่มสลับภาษา EN/TH
- ปุ่มสลับ Dark/Light theme
- ปุ่ม Download CV ที่หน้า Hero
- ส่วน About ที่มีตัวเลขสรุป (โปรเจกต์ที่สร้าง, เทคโนโลยีหลัก, ฝึกงาน, ภาษาโปรแกรมมิ่ง) และประวัติการศึกษา
- ส่วน Skills แบ่งเป็นหมวดหมู่ (Backend, Frontend, Database, DevOps, Tools, Soft Skills) พร้อม tag เทคโนโลยี

## สิ่งที่ตัดออก (ไม่ทำตามต้นแบบ)

- **ลิงก์/ปุ่ม GitHub ทั้งหมด** (ทั้งใน About และ Footer)
- **ปุ่ม Sign In / Sign Up** และระบบสมาชิก — ไม่จำเป็นสำหรับพอร์ตส่วนตัว
- **ฟอร์ม Contact ที่ล็อกอินก่อนถึงจะใช้ได้** — จะทำเป็นหน้าติดต่อธรรมดา ไม่ต้องล็อกอิน

## สิ่งที่เพิ่มเข้ามาใหม่ (ไม่มีในต้นแบบ)

### ส่วน "Projects" พร้อมหน้ารายละเอียดในตัวเว็บ

เว็บต้นแบบไม่มีส่วนโชว์โปรเจกต์แยก แต่คุณอยากได้ส่วนนี้เพิ่ม เพราะไม่อยากโชว์ GitHub และหลายโปรเจกต์ยังไม่พร้อม deploy จริง แนวทาง:

- หน้าแรกมีการ์ด "Projects" แสดงรายการโปรเจกต์ที่ทำแล้ว/กำลังทำ/วางแผนจะ host
- แต่ละการ์ดมีสถานะกำกับ เช่น `เสร็จแล้ว`, `กำลังพัฒนา`, `เตรียม deploy`
- กดที่การ์ดแล้วเข้าหน้ารายละเอียดโปรเจกต์ในเว็บเรา **เอง** (เช่น `/projects/project-name`) ไม่ใช่ลิงก์ออกไป GitHub
- หน้ารายละเอียดโปรเจกต์แต่ละอันมี: ชื่อ, คำอธิบาย, เทคโนโลยีที่ใช้, ภาพหน้าจอ/mockup, สถานะปัจจุบัน, และช่อง "ลิงก์ demo" ที่ยังว่างไว้ก่อน (ใส่ทีหลังตอน deploy จริงแล้ว)
- โครงสร้างนี้รองรับได้เลยแม้ตอนนี้ยังไม่มีโปรเจกต์ที่ deploy จริง — ใส่เป็น placeholder ไว้ก่อน แล้วค่อยเพิ่มลิงก์จริงทีหลังโดยไม่ต้องแก้โครงสร้างเว็บ

## โครงสร้างหน้าเว็บที่จะสร้าง

| หน้า | Path | เนื้อหา |
|---|---|---|
| หน้าแรก | `/` | Hero + About + Skills + Projects (แบบย่อ/preview) |
| รายละเอียดโปรเจกต์ | `/projects/[slug]` | ข้อมูลเต็มของแต่ละโปรเจกต์ |
| ติดต่อ | `/contact` | อีเมล, เบอร์โทร, LinkedIn/Facebook (ไม่มี GitHub, ไม่ต้องล็อกอิน) |

## ฟีเจอร์

- สลับภาษาไทย/อังกฤษ ทุกหน้า (จำค่าไว้)
- สลับ Dark/Light theme (จำค่าไว้)
- Responsive ทุกขนาดหน้าจอ
- ข้อมูลทั้งหมดแก้ได้จากไฟล์เดียว (`lib/content.ts`) เพื่อให้แก้ง่ายภายหลัง

## แผน Deploy ในอนาคต

Static site บน **Vercel** (ฟรี) — เหมาะกับพอร์ตโฟลิโอ ไม่ต้องมีเซิร์ฟเวอร์แยก เมื่อพร้อมค่อย connect repo กับ Vercel แล้ว deploy ได้ทันที
